# Roadmap

## MVP

- Landingpage
- Supabase Auth
- Geschützter App-Bereich
- Klasse erstellen oder per Code beitreten
- Rollen- und Berechtigungslogik
- Dashboard-Struktur
- Create/List-Flows für Termine, Aufgaben, Ressourcen, Ankündigungen und Abstimmungen
- Zod Schemas, TypeScript Types und Businesslogik
- Supabase-Schema und RLS-Konzept
- Vitest-Tests

## Nächste Schritte

1. Edit/Delete-Flows und Audit-Trail für alle Module ergänzen.
2. Poll-Ergebnisse optional mit Recharts visualisieren.
3. Supabase Storage für echte Datei-Uploads ergänzen.
4. Admin-Bereich für Mitglieder und Rollen ausbauen.
5. Notification-Tabelle mit In-App-Hinweisen verbinden.
6. Server Actions oder Route Handlers für progressive Enhancement ergänzen.
7. Audit- und Löschkonzept für produktive Datenschutzanforderungen vertiefen.

## Bewusst nicht im MVP

- Chat
- Fake Push Notifications
- Fake Uploads
- Demo-Schülerdaten im echten Nutzerflow
